# Beads Ticket Workflow Demo

This file demonstrates the complete beads ticket workflow.

## Workflow Phases

1. **CLAIM** - Update ticket to `in_progress`
2. **WORK** - Implement the changes
3. **COMMIT** - Create atomic commit with ticket reference
4. **LINT** - Run linting and fix any issues
5. **CLOSE** - Update ticket with summary and commit info
